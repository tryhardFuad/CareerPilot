## 4. Data Model (Supabase)

```sql
-- Core (RLS enforced on user_id = auth.uid())
profiles (id uuid pk = clerk_user_id, email, full_name, created_at)
cvs (id, user_id, file_url, version, raw_text, created_at)
cv_chunks (id, cv_id, user_id, content, embedding vector(1536), ordinality int)
jobs (id, user_id, title, company, source, url, description, posted_at, fetched_at)
applications (id, user_id, job_id, status enum['applied','interviewing','offer','rejected'], notes, applied_at, updated_at)
goals (id, user_id, title, target_date, progress int 0-100)
todos (id, user_id, goal_id null, title, due_at, completed bool, priority)
events (id, user_id, title, start_at, end_at, type enum['interview','deadline','goal'])
chat_sessions (id, user_id, title, created_at)
chat_messages (id, session_id, user_id, role, content, sources jsonb, tool_calls jsonb, created_at)

-- RLS: every table has policy `user_id = auth.uid()`
-- pgvector index: ivfflat on cv_chunks.embedding
```

## 5. Module Responsibilities

- **`lib/cv/ingester.ts`** — triggered by Inngest after upload: PDF/DOCX → text → 500-token chunks → OpenAI embeddings → upsert into `cv_chunks` with `user_id`.
- **`lib/agents/assistant.ts`** — `streamText` with tools: `retrieveCV`, `webSearch`, `saveApplication`. Loads last N messages from `chat_messages` for conversational memory.
- **`lib/agents/hunter.ts`** — accepts query + filters, calls `webSearch` or `jobBoards` tool, returns structured `{title, company, location, salary?, url, snippet}[]` rendered as `JobCard`s.
- **`lib/agents/fitScore.ts`** — combines: skill overlap (40%, Jaccard on normalized tokens), semantic similarity (40%, cosine), experience match (20%, years regex). Returns `{score, breakdown[], missingSkills[]}`.
- **`lib/memory/sessionStore.ts`** — persists every turn with sources + tool calls for audit + memory.
- **`components/tracker/KanbanBoard.tsx`** — `@dnd-kit/core` drag between 4 columns; optimistic update + Server Action.
- **`components/calendar/MonthCalendar.tsx`** — `react-day-picker` + custom event dots; to-do list filtered by selected date.

## 6. Implementation Phases

1. **Scaffold** — `create-next-app` (TS, App Router, Tailwind v4, Turbopack), install Clerk + Supabase + AI SDK, set brand tokens, base layout.
2. **Auth & Data** — Clerk middleware, `webhooks/clerk` to seed `profiles`, Supabase RLS policies.
3. **CV Pipeline** — Storage upload route, Inngest function, parser + chunker + embeddings, `cv_chunks` with pgvector.
4. **Assistant + RAG** — `/chat` page, streaming endpoint, `retrieveCV` tool, session memory, source citations.
5. **Hunter Agent** — `webSearch` + `jobBoards` tools, structured `JobCard` output.
6. **Fit Score** — scoring engine + UI gauge + explanation.
7. **Tracker Kanban** — `@dnd-kit`, 4 columns, status Server Actions.
8. **Calendar + To-Do** — events + todos tables, month view, goal-linked deadlines.
9. **Dashboard** — aggregate queries (apps/week, response rate, skills, roadmap %), Recharts.
10. **Marketing** — landing page with brand-dna hero, pricing, polish + tests.

## 7. Key Trade-offs / Decisions

- **Vercel AI SDK vs LangChain** — AI SDK wins: native streaming, first-class tool calling, smaller bundle, RSC-friendly. LangChain adds overhead we don't need.
- **Supabase pgvector vs Pinecone** — pgvector keeps auth + RLS + vectors in one Postgres row-level security boundary; cheaper at this scale. Pinecone only if we exceed ~1M chunks.
- **Inngest vs cron + queue** — Inngest gives retries, observability, and step functions for CV processing with zero infra.
- **Server Actions vs route handlers** — Server Actions for mutations (Kanban moves, to-do toggles); route handlers only for streaming chat + webhooks.
- **Conversational memory scope** — per-user session-scoped (last 20 messages in context window, full history in DB) — enough for memory demo, scales linearly.
